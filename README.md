## Структура проекта

```plaintext
project-root/
├── back/          # Бэкенд-приложение
├── front/         # Фронтенд-приложение
└── README.md      # Основная документация

    back — папка с бэкенд-кодом, где находятся все серверные компоненты.
    front — папка с фронтенд-кодом, который включает пользовательский интерфейс.
    README.md — данный файл, содержащий инструкции и описание проекта.

## Установка и запуск
### 1. Клонирование проекта

Сначала клонируйте репозиторий и перейдите в его корневую директорию.

git clone <https://github.com/BenchPressBoss/Registration-React-2.0.git>
cd project-root

### 2. Настройка окружения
Бэкенд

    Перейдите в папку back.


cd back

Создайте файл .env в папке back и укажите в нём переменные окружения:

plaintext

DATABASE_URL=postgresql://<username>:<password>@localhost:<port>/<database>?schema=public
JWT_SECRET=<ваш секретный ключ>

Убедитесь, что база данных PostgreSQL настроена и доступна. Если нужно, создайте базу данных вручную перед миграциями.

Выполните миграции Prisma, чтобы создать таблицы:

bash

npx prisma migrate dev --name init

### Запустите сервер:

    npm install
    npm run dev

## Фронтенд

    Перейдите в папку front.

    cd front

## Установите зависимости и запустите приложение:

bash

    npm install
    npm run dev


    Авторизация и регистрация: Реализовано с использованием JWT. Пользователь может зарегистрироваться и получить токен для доступа к защищённым ресурсам.
    Настройка базы данных: Используется PostgreSQL и Prisma для взаимодействия с базой данных. В папке back есть папка prisma, где хранятся схемы для миграций базы.
    Переменные окружения:
        DATABASE_URL: строка подключения к базе данных.
        JWT_SECRET: секретный ключ для генерации и проверки JWT-токенов.

## Фронтенд

    UI и маршрутизация: Фронтенд построен с использованием React и содержит компоненты для страниц авторизации, чата и оплаты.
    Работа с данными: Используется React Query для работы с данными и кеширования, а также QueryClientProvider для доступа к клиенту данных в любом компоненте.
    Уведомления: Компонент Toaster используется для показа уведомлений (ошибки, успешные действия и т.д.).

## Инструменты для разработки

    React Query Devtools: Инструмент для отладки работы с запросами на фронтенде. Включён с параметром initialIsOpen={false}, чтобы не отображаться по умолчанию.
    Prisma CLI: используется для миграций базы данных и управления схемой.

## Примечания

    Убедитесь, что файл .env настроен корректно для работы с базой данных и JWT.
    Проверяйте настройки CORS для корректного взаимодействия фронтенда и бэкенда.
    README.md находится в корне проекта, что позволяет быстро получить доступ к документации и начать работу.
```
